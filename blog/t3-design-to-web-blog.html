A responsive site is where the webpage reacts dependent on the size of the screen that it is being viewed upon. For example, if a website is being viewed upon a mobile phone, the content needs to be as accessible as it is upon a desktop screen, and everything in between.

Mobile first design is such that the code are written primarily for mobile, the smallest screen (with in theory the least processing power). The code is written so that as screen sizes increase, the layout of the site will adapt to appropriately fit the screen. Mobile first is important for a number of reasons, primarily due to the fact that an increasing percentage of people primarily use their mobile to access the internet. Mobile first means designers priority is to get all relevant content in the mobile functionality first, then adapt that to fit a larger screen rather than vice versa.

A framework is designed to provide some core structure and design elements to a website so a developer doesn't need to start from scratch when building a website. In this regard, it can save a lot of time. Their primary disadvantage is when it comes to beginners, when frameworks can help create a nice looking website, whilst obscuring some blind spots from the student's knowledge base.

A wireframe allows us to mock up the fundamental structure quickly and easily, and immediately get feedback and make changes before even starting coding.
